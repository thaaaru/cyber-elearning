> # 3. Information Gathering &amp; Reconnaissance
> 
> ## 3.1. Reconnaissance Methodologies
> - [ ] **Differentiate between [[Passive vs. Active Reconnaissance]]**
>     - [ ] **Passive**: Gathering information without directly engaging with the target&#x27;s systems (e.g., search engines, public records). Goal: Remain undetected.
>     - [ ] **Active**: Directly probing target systems (e.g., port scanning, server interaction). Goal: Gather detailed technical data. Carries a risk of detection.
> 
> ## 3.2. Foundational OSINT Techniques
> - [ ] **Master Search Engine Reconnaissance: [[Google Dorking]]**
>     - [ ] Understand key operators: &#x60;site:&#x60;, &#x60;inurl:&#x60;, &#x60;intitle:&#x60;, &#x60;filetype:&#x60;.
>     - [ ] Practice finding sensitive information: Login pages, exposed documents (&#x60;.pdf&#x60;, &#x60;.xls&#x60;), error messages.
>     - [ ] Explore the Google Hacking Database (GHDB) for advanced queries.
> - [ ] **Extract Domain and IP Information**
>     - [ ] **Perform [[WHOIS]] Lookups**:
>         - [ ] Identify registrant name, contact details, domain registration/expiry dates, and nameservers.
>     - [ ] **Conduct [[DNS Enumeration]]**:
>         - [ ] Use &#x60;nslookup&#x60; and &#x60;dig&#x60; for manual queries.
>         - [ ] Identify key DNS records: &#x60;A&#x60; (Host), &#x60;MX&#x60; (Mail), &#x60;NS&#x60; (Name Server), &#x60;TXT&#x60; (SPF, DKIM).
> 
> ## 3.3. Infrastructure and People Mapping
> - [ ] **Map Publicly Exposed Infrastructure**
>     - [ ] **Use [[Shodan]]**: &quot;The search engine for hackers.&quot;
>         - [ ] Find devices by type (e.g., webcams, routers), technology (e.g., Apache), or organization.
>         - [ ] Identify open ports and vulnerable service banners.
>     - [ ] **Use [[Censys]]**:
>         - [ ] Discover hosts, websites, and certificates associated with a target.
> - [ ] **Gather Employee Data Responsibly**
>     - [ ] Use LinkedIn to identify roles, responsibilities, and potential contacts.
>     - [ ] Correlate names with email address formats (e.g., &#x60;first.last@company.com&#x60;).
>     - [ ] *Ethical Consideration*: Emphasize the responsible and legal limits of this activity.
> 
> ## 3.4. Technical Footprinting
> - [ ] **Perform [[Subdomain Enumeration]]**
>     - [ ] Use online tools and search engines.
>     - [ ] Leverage Certificate Transparency logs.
>     - [ ] Use command-line tools like &#x60;[[Sublist3r]]&#x60; or &#x60;amass&#x60;.
> - [ ] **Conduct [[Technology Stack Identification]]**
>     - [ ] Use browser extensions like &#x60;[[Wappalyzer]]&#x60; or &#x60;BuiltWith&#x60;.
>     - [ ] Identify the target&#x27;s CMS (e.g., WordPress), web server (e.g., Nginx), JavaScript frameworks (e.g., React), and analytics tools.
> 
> ## 3.5. Automated OSINT Collection
> - [ ] **Automate Data Gathering with [[theHarvester]]**
>     - [ ] Collect emails, subdomains, hosts, and open ports from multiple public sources (e.g., Google, Bing, Shodan).
> - [ ] **Visualize Connections with [[Maltego]]**
>     - [ ] Understand the graph-based approach to link disparate pieces of public information.
>     - [ ] Perform &quot;transforms&quot; to pivot from one data point (like a domain) to related entities (like DNS names, IP addresses, or employee names).
> 
> ## 3.6. Documentation and Reporting
> - [ ] **Develop a [[Reconnaissance Reporting]] Template**
>     - [ ] **Section 1: Executive Summary**: High-level overview of findings.
>     - [ ] **Section 2: Scope**: Clearly define the target domains/IPs.
>     - [ ] **Section 3: Findings**: Detail all discovered information (domains, IPs, technologies, emails).
>     - [ ] **Section 4: Evidence**: Provide screenshots, command outputs, and links.
>     - [ ] **Section 5: Potential Attack Vectors**: Hypothesize how this information could be used by an attacker.
